## Highlights

This patch release tightens the internal analysis layer, improves cache behavior for long-lived sessions, and cleans up project bookkeeping without changing the public MCP tool surface.

## Improved

- Bounded the FDIC query cache to avoid unbounded memory growth during long-lived stdio or HTTP sessions.
- Added warnings when batched FDIC analysis queries appear truncated at the API's 10,000-record response cap.
- Extracted shared analysis and peer-group batching helpers into a common module to reduce duplication.
- Extracted reusable financial metric helpers into a shared module so peer-group and future analytics paths can reuse one implementation.
- Expanded MCP HTTP coverage for failures, history, SOD, and annual summary tools.
- Clarified maintainer guidance in `AGENTS.md` and normalized stale completed checklist items in `tasks/todo.md`.

## Notes

- The npm package remains available as `fdic-mcp-server`.
- The hosted HTTP endpoint remains `https://bankfind.jflamb.com/mcp`.
- This release is intended to exercise the tagged publish and deployment workflow paths after the recent workflow hardening changes.
